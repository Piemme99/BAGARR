Le Timer

Un chrono qui n'avance que si tu le pousses. Démystifier from_seconds, tick, just_finished… et le fameux .0.

01L'idée clé : un Timer n'avance pas tout seul

Un Timer ne « tourne » pas en tâche de fond. C'est un compteur de temps écoulé que toi dois faire avancer à chaque frame en lui donnant le temps de la frame. Image : un chronomètre dont l'aiguille ne bouge que si tu appuies dessus à chaque image.

Trois gestes :

// 1. Créer : durée + mode
Timer::from_seconds(1.0, TimerMode::Repeating)

// 2. Chaque frame : l'avancer du temps écoulé
timer.tick(time.delta());       // time.delta() = durée de CETTE frame

// 3. Réagir quand il sonne
if timer.just_finished() { /* … */ }

02Simulateur : appuie sur « frame »

Chaque clic simule une frame : ça tick le timer de ~0,25 s. Observe just_finished() qui ne passe au vert qu'une seule frame, puis (mode Repeating) le timer repart de zéro.

elapsed : 0.00 / 1.00 s just_finished() = false
—
finished vs just_finished finished() reste vrai tant que le timer a dépassé sa durée (utile pour un Once). just_finished() n'est vrai que la frame précise où il vient de sonner — c'est ce qu'on veut pour « faire spawn un ennemi une fois par seconde », pas 60 fois.

03Once vs Repeating

TimerMode::Once : sonne une fois puis reste « fini » (se bloque à la durée). TimerMode::Repeating : se réarme tout seul après chaque sonnerie (repart de 0). Pour un spawn périodique → Repeating.

04Le fameux .0 — c'est du Rust, pas du Bevy

On range le timer dans une ressource :

#[derive(Resource)]
struct EnemySpawnTimer(Timer);   // ← un "tuple struct" : un champ, sans nom

EnemySpawnTimer(Timer) est un tuple struct : une struct dont les champs n'ont pas de nom, juste un numéro. Le premier (et seul) champ s'appelle donc .0 :

spawn_timer.0.just_finished()
//          └─ .0 = le Timer à l'intérieur du wrapper

Compare avec une struct à champs nommés :

struct EnemySpawnTimer { timer: Timer }   // alors ce serait spawn_timer.timer

Donc .0 = « le 1ᵉʳ champ » d'un tuple struct. On emballe le Timer dans un type nommé EnemySpawnTimer uniquement pour en faire une Resource distincte (Bevy identifie les ressources par leur type — deux timers = deux types différents).

La ligne condensée Comme tick renvoie une référence au timer, on peut enchaîner : 
if spawn_timer.0.tick(time.delta()).just_finished() { /* spawn */ }
« avance le chrono, et s'il vient de sonner, agis » — en une ligne.

✦Récap

Un Timer = un compteur que tu fais avancer avec tick(time.delta()) chaque frame ; just_finished() = « il vient de sonner cette frame » (une seule fois) ; Repeating le réarme tout seul. On l'emballe dans un tuple struct #[derive(Resource)] pour en faire une ressource — d'où le .0 qui accède au Timer dedans.
