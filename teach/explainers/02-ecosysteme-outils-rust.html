L'écosystème d'outils Rust

rustup, cargo, rustc et crates.io — qui fait quoi, et pourquoi rustup update a débloqué ton build Bevy.

1 · Qui fait quoi 2 · Comment ils s'enchaînent 3 · rustup 4 · cargo & crates.io 5 · Les versions (semver)

Quand tu as installé Rust, tu n'as pas installé un programme mais une petite famille d'outils qui se répartissent le travail. Comprendre qui fait quoi évite de chercher au mauvais endroit — et explique pourquoi l'erreur de tout à l'heure se réglait avec rustup, pas avec cargo.

01Qui fait quoi

🎛️
rustup
Gère quelles versions de Rust sont installées et laquelle est active. Installe, met à jour, bascule entre stable/beta/nightly.
≈ nvm (Node) · pyenv (Python)
📦
cargo
Orchestre ton projet : télécharge les dépendances, lance la compilation, exécute, teste. C'est l'outil que tu tapes 99% du temps.
≈ npm + webpack
⚙️
rustc
Le compilateur lui-même. Traduit le code en machine natif. Appelé automatiquement par cargo — tu l'invoques rarement à la main.
≈ le moteur qui tourne sous le capot
🌐
crates.io
Le registre public des bibliothèques Rust (les « crates »). C'est de là que cargo télécharge Bevy et ses centaines de dépendances.
≈ le registre npm / PyPI
Le déclic de tout à l'heure Ton erreur disait « bevy@0.19 requires rustc 1.95 ». C'est un problème de version de Rust → domaine de rustup, d'où rustup update stable. Si ç'avait été « dépendance introuvable », c'eût été un problème de projet → domaine de cargo.
Tu veux essayer une fonctionnalité expérimentale qui n'existe que dans Rust nightly. Quel outil utilises-tu ?

02Comment ils s'enchaînent

Quand tu tapes cargo build, voici le ballet :

rustupa fourni la toolchain (rustc + cargo + std)
→
cargolit Cargo.toml, résout les versions
→
crates.iocargo télécharge les crates manquantes
→
rustccargo l'appelle pour compiler chaque crate

Autrement dit : rustup pose le terrain de jeu, cargo est le chef d'orchestre, il va chercher les partitions sur crates.io, et fait jouer rustc. Tu ne parles presque jamais directement à rustc.

03rustup & la notion de toolchain

Une toolchain = un paquet cohérent installé ensemble : rustc + cargo + la bibliothèque standard + quelques composants. Elle est identifiée par un canal et une version :

CanalC'est quoiPour qui
stableversion fiable, publiée toutes les ~6 semainestoi (défaut)
betala prochaine stable, en testtester en avance
nightlybuild quotidien, fonctions expérimentalesfeatures de pointe

Chaque toolchain a des composants : certains obligatoires (rustc), d'autres optionnels comme clippy (linter) ou rustfmt (formateur automatique).

Sources : The rustup book · Toolchains

Essaie ça Inspecte ta toolchain (après le rustup update) : 
rustup show            # toolchains installées + active
rustc --version        # doit afficher ≥ 1.95 maintenant
rustup component add clippy   # ajoute le linter officiel (très utile en apprentissage)

04cargo & crates.io

Cargo s'appuie sur deux fichiers dans ton projet — ne les confonds pas :

FichierRôleTu l'édites ?
Cargo.tomlce que tu veux : nom, deps + contraintes de version, profilsoui, à la main
Cargo.lockce que cargo a résolu : les versions exactes choisies, figéesnon, généré

Le lockfile (Cargo.lock) garantit que ta machine et celle d'un collègue compilent exactement les mêmes versions, même si une nouvelle sort entre-temps. Pour un binaire (notre cas), on le versionne dans git.

Sources : The Cargo Book · Dependency Resolution

Essaie ça Vois l'arbre de dépendances que Bevy tire depuis crates.io : 
cargo tree            # affiche toutes les crates, en arbre
cargo tree | wc -l    # … et compte-les. Prépare-toi à sourire.

05Les versions : bevy = "0.19" veut dire quoi ?

Une chaîne de version dans Cargo.toml n'est pas une version figée mais une contrainte (un intervalle). Par défaut, écrire "0.19" équivaut à "^0.19" : le caret, qui autorise les mises à jour compatibles selon le semver. La règle : on peut bouger tout ce qui est à droite du premier chiffre non-nul, jamais ce chiffre-là ni ce qui est à gauche.

^0.19.0
major
0
minor
19
patch
0

Pour bevy = "0.19", le premier chiffre non-nul est le 19 (le major est 0). Donc cargo prendra n'importe quel 0.19.x (dernier patch), mais jamais 0.20 — une montée de mineure en 0.x est considérée comme potentiellement cassante. C'est pour ça qu'on ne saute pas magiquement en 0.20 tout seul.

Source : Cargo Book — Specifying Dependencies

Avec bevy = "0.19", cargo peut-il choisir la version 0.19.4 si elle sort ? Et 0.20.0 ?

✦Récap

rustup gère tes versions de Rust (d'où rustup update pour la MSRV de Bevy) ; cargo pilote ton projet et va chercher les crates sur crates.io ; rustc compile, appelé par cargo ; Cargo.toml dit ce que tu veux, Cargo.lock fige ce qui a été résolu ; et "0.19" est une contrainte caret qui accepte tout 0.19.x mais pas 0.20.
