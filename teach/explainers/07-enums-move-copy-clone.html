Enums, move, Copy & Clone

Deux piliers de Rust : modéliser un choix fini avec un enum, et comprendre pourquoi indexer un tableau t'a forcé à #[derive(Copy)].

1 · Les enums 2 · Variantes avec données 3 · Ownership & move 4 · Copy vs Clone 5 · derive & ton Edge

01Un enum = « exactement une valeur parmi un ensemble fini »

Un enum définit un type dont une valeur est exactement l'une de variantes nommées. Ton Edge :

enum Edge { Top, Bottom, Left, Right }
// une valeur de type Edge est SOIT Top, SOIT Bottom, SOIT Left, SOIT Right — jamais deux, jamais autre chose

La force : le compilateur connaît toutes les variantes. Un match qui les couvre toutes est exhaustif — pas besoin de _, et si tu ajoutes une variante plus tard, le compilateur t'oblige à traiter le nouveau cas partout. C'est ce que tu as gagné en passant de match sur i32 (qui exigeait un _) à match edge.

Source : Rust Book — Ch.6 Enums

02Les variantes peuvent porter des données

Contrairement à un enum C (juste des entiers nommés), une variante Rust peut contenir des données, différentes selon la variante :

enum Attaque {
    Poing,                       // rien
    Projectile(Vec2),            // une direction
    Zone { rayon: f32, degats: u32 },  // des champs nommés
}

Et match extrait ces données en même temps qu'il distingue la variante :

match attaque {
    Attaque::Poing => { /* … */ }
    Attaque::Projectile(dir) => { // dir est le Vec2 }
    Attaque::Zone { rayon, degats } => { // rayon et degats dispo }
}
Tu en utilises déjà ! Option<T> et Result<T, E> sont des enums de la std : 
enum Option<T> { Some(T), None }
enum Result<T, E> { Ok(T), Err(E) }
Ton let Ok(mut transform) = query.single_mut() else {…}, c'est un match sur l'enum Result qui extrait le T de la variante Ok. Tu faisais du pattern matching sans le savoir.

03Ownership & move : pourquoi indexer « sort » une valeur

Règle d'or de Rust : chaque valeur a un seul propriétaire. Quand tu fais let b = a;, par défaut Rust déplace (move) la valeur de a vers b — et a devient inutilisable (pour éviter deux propriétaires du même truc).

Or un tableau possède ses éléments. edges[i] essaie de sortir l'élément « à la valeur » → ça laisserait un trou dans le tableau (un move partiel) → Rust refuse (c'était ton erreur E0508). Sauf si le type est Copy (§4).

Source : Rust Book — Ch.4.1 Ownership

04Copy vs Clone : dupliquer, implicite ou explicite

Certains types sont triviaux à dupliquer (juste recopier des octets, aucun coût caché) : les entiers, bool, char, les flottants, et les enums/structs qui ne contiennent que ça. Pour eux existe le trait Copy :

Lien entre les deux : Copy exige Clone (tout ce qui est copiable est aussi clonable). D'où le duo #[derive(Clone, Copy)]. Un type qui contient du tas (String, Vec, un Handle…) ne peut pas être Copy (dupliquer bêtement les octets casserait la propriété du tas) — au mieux Clone.

05derive : et donc, ton Edge

Un trait = un ensemble de comportements qu'un type peut fournir. Écrire l'implémentation à la main est fastidieux pour des traits « mécaniques » → #[derive(...)] demande au compilateur de générer l'implémentation pour toi.

#[derive(Debug, Clone, Copy)]
enum Edge { Top, Bottom, Left, Right }

Pourquoi ces trois :

  • Copy → indexer edges[i] copie la variante au lieu de la move hors du tableau (règle ton erreur). Un enum sans données est trivialement copiable.
  • Clone → requis par Copy (obligatoire de l'avoir aussi).
  • Debug → permet d'afficher avec {:?} (pratique pour déboguer).
Peux-tu mettre #[derive(Copy)] sur struct Joueur { nom: String, pv: u32 } ?
Essaie ça (mental) Regarde tes types de BAGARR : Vec2, Vec3, Color sont Copy (que des nombres) — c'est pour ça que tu les passes partout sans .clone(). À l'inverse, un Handle<Mesh> n'est que Clone (il compte des références) : tu le clone()rais si besoin, il ne se copie pas tout seul.

✦Récap

Un enum = une valeur parmi un ensemble fini de variantes (qui peuvent porter des données) ; match les distingue de façon exhaustive (d'où « pas de _ »). Par défaut, assigner/indexer déplace (move) une valeur et invalide l'original ; les types « plats » marqués Copy sont copiés à la place (original conservé), et Clone duplique explicitement (potentiellement cher). Ton Edge, enum sans données, dérive Copy pour que edges[i] copie au lieu de move.
